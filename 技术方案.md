# 基于 C++ 与 OpenGL 的本地太阳系实时渲染 — 技术方案

## 1. 项目概述

### 1.1 项目目标

构建一个**不依赖商业游戏引擎**的本地太阳系实时渲染程序，使用 C++17 与 OpenGL 4.6 Core Profile 完成渲染管线，实现太阳、八大行星的三维可视化、公转自转模拟、自由摄像机飞行与基础光照系统。

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| **无商业引擎** | 不引入 Unity / Unreal 等商业引擎，全部手写渲染管线 |
| **可移植性** | 所有第三方图形库（GLFW、GLAD、GLM、stb_image 等）以源码或静态库形式保存在项目 `thirdparty/` 目录下 |
| **可扩展性** | 模块化架构，预留大气散射、LOD、多线程资源管理等扩展点 |
| **学习导向** | 代码结构清晰，注释适度，适合作为现代图形学学习参考 |

## 4. 渲染管线设计

### 4.1 整体流程

```
┌─────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌───────────┐
│ 输入处理 │ → │ 场景更新  │ → │ 阴影 Pass │ → │ 主渲染   │ → │ UI 叠加   │
│ (Input)  │   │ (Time +  │   │ (Phase 2) │   │ (Forward  │   │ (ImGui)   │
│           │   │  Orbit)  │   │           │   │  Render)  │   │           │
└─────────┘   └──────────┘   └──────────┘   └──────────┘   └───────────┘
```

### 4.2 着色器设计

#### 行星着色器 (planet.vert / planet.frag)

- **Phong 光照模型**（Ambient + Diffuse + Specular）
- 单点光源（太阳位置），方向由太阳中心指向片元
- 法线贴图支持（预留）
- 纹理采样（漫反射贴图）

```glsl
// 光照计算核心（片段着色器）
vec3 ambient  = ambientStrength * lightColor * texture(diffuseMap, TexCoords).rgb;
vec3 diffuse  = diff * lightColor * texture(diffuseMap, TexCoords).rgb;
vec3 specular = spec * lightColor * texture(specularMap, TexCoords).rgb; // 可选
vec3 result   = (ambient + diffuse + specular) * attenuation;
```

#### 太阳着色器 (sun.vert / sun.frag)

- **自发光 + Bloom 预处理**（后期全屏后处理叠加）
- 动态噪点扰动模拟日面活动（可选，Phase 2）
- HDR 颜色输出（`vec3` > 1.0）

#### 天空盒着色器 (skybox.vert / skybox.frag)

- Cubemap 采样，深度写入禁用，最后绘制

### 4.3 Forward Rendering

初期采用正向渲染 (Forward Rendering)，单 Pass 完成所有绘制：

1. 绑定天空盒 Cubemap → 绘制天空盒（深度测试改为 `GL_LEQUAL`）
2. 绑定太阳 Shader → 绘制太阳
3. 绑定行星 Shader → 依次绘制八颗行星（状态切换最小化：按 Shader 分组）

## 5. 天体系统设计

### 5.1 数据结构

```cpp
// 天体参数（初始化时配置）
struct CelestialParams {
    std::string name;           // 名称
    float       radius;         // 渲染半径（km，内部缩放后使用）
    float       orbitRadius;    // 轨道半长轴（km）
    float       orbitSpeed;     // 公转角速度（rad/s，会被时间缩放影响）
    float       rotationSpeed;  // 自转角速度（rad/s）
    float       axialTilt;      // 轴倾角（度）
    glm::vec3   orbitNormal;    // 轨道面法线
    std::string texturePath;    // 漫反射贴图路径
};

class CelestialBody {
protected:
    CelestialParams m_params;
    Mesh            m_mesh;       // 球体 mesh（共享引用或实例）
    glm::mat4       m_modelMatrix;
    float           m_orbitAngle  = 0.0f; // 当前轨道角度
    float           m_selfAngle   = 0.0f; // 当前自转角度
public:
    virtual void update(float deltaTime);
    virtual void draw(Shader& shader);
};

class Planet : public CelestialBody { /* 使用 Phong 光照 */ };
class Star   : public CelestialBody { /* 自发光 + 点光源 */ };
```

### 5.2 坐标变换

```
局部模型空间 → 自转矩阵(R_self) → 轨道平移(T_orbit) → 公转矩阵(R_orbit) → 世界空间 → 视图矩阵(V_camera) → 投影矩阵(P)
```

每个天体的 model matrix 计算：

```
M_model = T_orbit(orbitRadius, 0, 0) * R_orbit(orbitAngle, orbitNormal) * R_axialTilt * R_self(selfAngle, axisY)
```

### 5.3 缩放策略

真实太阳系尺度无法直接在 GPU 上以 1:1 渲染（双精度问题 + 行星过小），采用**对数缩放**或**分段缩放**：

- **距离缩放**: 轨道半径缩小 10⁻⁴ ~ 10⁻⁵
- **尺寸缩放**: 天体半径放大 10² ~ 10³（确保可见性）
- 两套缩放独立控制，通过 ImGui 面板实时调节

### 5.4 轨道渲染

- 使用 `GL_LINE_LOOP` 绘制圆形/椭圆轨道线
- 每个行星一个 Orbit 对象，分辨率 256 段
- 半透明颜色（alpha < 1.0），在行星之后绘制

### 6.2 数学实现

```cpp
// 欧拉角 + 方向向量法
void Camera::updateVectors() {
    glm::vec3 front;
    front.x = cos(glm::radians(m_yaw)) * cos(glm::radians(m_pitch));
    front.y = sin(glm::radians(m_pitch));
    front.z = sin(glm::radians(m_yaw)) * cos(glm::radians(m_pitch));
    m_front   = glm::normalize(front);
    m_right   = glm::normalize(glm::cross(m_front, m_worldUp));
    m_up      = glm::normalize(glm::cross(m_right, m_front));
}
```

## 7. 光照系统

### 7.1 Phase 1: 基础光照

- **唯一光源**: 太阳作为点光源（Point Light），位置 = 世界原点
- **光照模型**: Blinn-Phong
  - Ambient: 统一环境光，避免背光面全黑（模拟微弱星光）
  - Diffuse: `max(dot(N, L), 0)`
  - Specular: `pow(max(dot(N, H), 0), shininess)`
- **衰减**: 线性衰减，归一化到太阳系尺度

### 7.2 Phase 2: 扩展光照（后续）

- **阴影映射 (Shadow Mapping)**: 以太阳为视点的深度图，处理行星遮挡
- **HDR + 泛光 (Bloom)**: 太阳自发光部分 > 1.0，高斯模糊后叠加
- **大气散射**: Rayleigh / Mie 散射（地球、土卫六等有大气层的天体）

## 8. 资源管理

### 8.1 Mesh 管理

- 球体 mesh 程序化生成，参数：半径、经线数 (sectors)、纬线数 (stacks)
- 所有行星共享同一个单位球 mesh 实例，通过 model matrix 的 scale 变换大小
- 太阳使用单独的细分更高的球体 mesh

### 8.2 纹理管理

```cpp
class TextureCache {
    std::unordered_map<std::string, Texture> m_cache;
public:
    Texture& load(const std::string& path);
    Texture& get(const std::string& path);
    void    clear();
};
```

- 纹理加载使用 stb_image，支持 JPEG/PNG
- 内部缓存避免重复加载
- 支持 sRGB 自动转换（`GL_SRGB` / `GL_SRGB_ALPHA`）

### 8.3 Shader 管理

- Shader 从文件读取，编译错误时输出完整日志
- 利用 `#include` 机制（手动解析或 GL_ARB_shading_language_include）共享公共 uniform 块和函数

## 9. 主循环设计

```cpp
int main() {
    Window window(1920, 1080, "Solar System");
    Input  input(window);
    Time   time;

    SolarSystem solarSystem;
    Renderer    renderer;

    // 主循环
    while (!window.shouldClose()) {
        time.tick();                    // 更新 deltaTime
        input.poll();                   // 处理输入事件

        solarSystem.update(time);       // 更新天体位置 / 自转
        camera.update(input, time);     // 更新摄像机

        renderer.clear();
        renderer.drawSkybox(camera);
        renderer.drawSolarSystem(solarSystem, camera, time);

        // ImGui 调试面板
        #ifdef DEBUG
        renderer.drawUI(solarSystem, camera, time);
        #endif

        window.swapBuffers();
    }
}
```

### 9.1 时间系统

```cpp
class Time {
    float m_deltaTime;      // 帧间隔 (s)
    float m_timeScale;      // 时间缩放因子 (0 = 暂停, 1 = 实时, 100 = 快速)
    float m_elapsedTime;    // 程序总运行时间
public:
    void tick();            // 每帧调用
    float getDeltaTime() const;    // = m_deltaTime * m_timeScale
    float getElapsedTime() const;
    void  setTimeScale(float s);
};
```

时间缩放允许用户加速/减速/暂停天体运动，便于观察。

## 10. 扩展路线图

### 10.1 Phase 2: 大气散射

- 参考 Bruneton & Neyret (2008) 预计算大气散射
- LUT 纹理存储透射率，实时采样
- 仅对有大气的天体（地球、土卫六）启用
- Shader 独立为 `atmosphere.vert / .frag`

### 10.2 Phase 3: LOD 与性能优化

- **天体 LOD**: 远距离行星使用低细分球体，近距离动态增加细分
- **纹理 Mipmap**: 使用 `glGenerateMipmap` + 各向异性过滤
- **实例化渲染**: 小行星带使用 `glDrawElementsInstanced`
- **视锥剔除**: 每帧检测天体包围球与视锥的相交

### 10.3 Phase 4: 多线程资源管理

- **资源加载线程**: 纹理从磁盘异步加载，主线程不阻塞
- **OpenGL 上下文共享**: 副线程持有共享 context，上传纹理数据
- **任务调度**: 使用 `std::async` / 线程池管理后台任务
- **双缓冲资源表**: 加载完成的资源先写入 pending 表，主线程在帧开始时 swap

### 10.4 Phase 5: 程序化星球生成

- **LOD 四叉树球体**: 基于 `gl_Position` 动态细分
- **噪声高度场**: Perlin / Simplex 噪声生成地形
- **颜色映射**: 高度 → 颜色渐变（海平面、草地、岩石、雪线）
- **GPU-Driven**: 在 Compute Shader 中生成顶点数据，直接写入 VBO
