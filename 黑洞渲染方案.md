# 黑洞渲染方案

本文档记录黑洞场景当前状态，并重点规划后续更新方向。当前项目已经完成从“可切换黑洞场景”到“可讲解的实时黑洞近似渲染”的第一轮闭环，后续重点将从视觉近似逐步推进到更真实的物理近似、更好的教学表达和更稳定的性能分层。

## 当前状态

### 已完成内容

- `SceneCatalog` 已启用 `Black Hole` 场景
- `BlackHoleScene` 已接入主循环，且采用懒加载，默认太阳系启动不编译黑洞 shader
- 事件视界使用纯黑球体渲染，保持吸光外观
- 吸积盘使用圆环 mesh + shader 温度渐变 + 旋转动画
- 引力透镜使用屏幕空间后处理 pass
- 透镜 pass 已加入低成本 Schwarzschild 风格光线步进，可从相机 ray 近似弯折并判断事件视界捕获
- 光子球附近有可调亮环
- 吸积盘已有近似真实感：
  - Doppler 增亮与蓝/红色偏
  - 湍流纹理
  - 盘面厚度视觉
  - 远近侧遮挡
  - 自旋导致的透镜非对称
- 黑洞控制面板已有：
  - 参数 preset
  - Debug preset
  - 镜头 preset
  - 标签和说明面板
- 启动阶段已有加载画面，避免黑屏等待

## 当前渲染管线

1. 绘制 skybox 星场
2. 绘制太阳系或黑洞场景
3. 黑洞场景中先绘制吸积盘，再绘制事件视界
4. 如果当前场景是黑洞，`Renderer` 执行透镜 pass：可在快速径向透镜和 Schwarzschild 风格 ray marching 之间切换
5. bloom 提取高亮区域
6. HDR composite 输出到默认 framebuffer
7. 绘制教学标签、说明面板和 ImGui 控制面板

当前管线的优点是简单、稳定、实时。F1 已经加入从相机发出 view ray 的近似步进，但仍不是完整 geodesic 求解，复杂遮挡、背面盘面多重成像和真实 Kerr 光线绕行仍属于后续阶段。

## 后续更新总目标

后续黑洞场景不只是“更好看”，而是要逐步变成一个可演示、可解释、可调试、可扩展的黑洞教学渲染模块。

目标分为四类：

- **真实感**：从屏幕空间扭曲升级到更接近 Schwarzschild / Kerr 的光线近似
- **可讲解**：让用户能理解事件视界、光子球、吸积盘、Doppler、透镜等概念
- **可调试**：每个视觉组成都有 debug view 和参数 preset
- **可扩展**：后续能接入喷流、星场透镜、双黑洞、潮汐撕裂事件等新场景

## 阶段 F：物理透镜升级

### F1：Schwarzschild 近似光线步进（已完成第一版）

当前实现已经在透镜 shader 中加入低成本 ray marching：

- 从相机发出 view ray
- 在黑洞附近根据距离施加弯曲
- 累积若干步后采样星场或吸积盘
- 根据最近距离判断是否落入事件视界

第一版不追求严格解，目标是得到更自然的绕行和阴影边界，同时保留快速径向透镜作为兼容 fallback。

建议增加参数：

```ini
[Lensing]
quality = medium
raySteps = 48
stepScale = 0.85
massStrength = 1.0
captureRadius = 12.0
```

当前已接入的运行时参数包括：

- `Schwarzschild March`
- `Ray Steps`
- `Step Scale`
- `Mass Strength`
- `Capture Radius`
- `Ray Bend / Ray Closest` debug view

### F2：Kerr 自旋近似

在 Schwarzschild 近似稳定后，加入自旋导致的 frame dragging：

- 自旋方向影响切向偏折
- 光子环左右亮度不对称
- 黑洞阴影中心轻微偏移
- 高自旋 preset 更明显地展示非对称

注意这仍然可以是视觉近似，不必立即实现完整 Kerr metric。

### F3：透镜质量分档

需要至少三个质量档：

- `Fast`：当前屏幕空间透镜，适合集显
- `Balanced`：少量 ray steps，适合主流显卡
- `High`：更多步数和更精细采样，适合截图/演示

UI 中应显示当前质量档和估计步数，方便教学时解释性能成本。

## 阶段 G：吸积盘真实感升级

### G1：盘面三维厚度

当前厚度主要由 shader 伪造。后续可以改成双层或体积近似：

- 上下两层 disk shell
- 侧面 rim mesh
- 根据视角显示盘面边缘厚度
- 内圈更厚、更亮，外圈更薄、更冷

这会让边缘视角更可信，也能给“吸积盘不是纸片”提供教学依据。

### G2：自遮挡与多重成像

后续应把吸积盘分成前景盘面、背面盘面和被透镜抬起的盘面：

- 前景盘面遮挡部分事件视界
- 背面盘面被透镜弯到黑洞上方
- 近黑洞内缘形成更强亮环
- 不同 debug view 分别显示 front/back/lensed contribution

### G3：发射模型

当前温度是简单径向渐变。后续可加入更物理的 emissivity：

- `T(r) ~ r^-3/4` 的近似温度分布
- 内稳定圆轨道 ISCO 与自旋关联
- 内圈温度和亮度不再只是手调参数
- 外圈尘埃/冷气体颜色更红、更暗

### G4：湍流与磁流体视觉

无需上完整 MHD 模拟，但可以加入更好的视觉结构：

- 稳定的程序化噪声贴图
- 螺旋臂状亮带
- 时间连续的流动纹理
- 可调“安静盘 / 湍流盘 / 爆发盘”preset

## 阶段 H：喷流、热冕和环境

### H1：相对论喷流

喷流能让黑洞场景更具辨识度：

- 沿自旋轴发出双向喷流
- 使用半透明 cone / ribbon / particle
- 颜色从白蓝核心过渡到紫蓝外缘
- 可调 opening angle、length、intensity

教学点：

- 喷流方向和吸积盘法线/黑洞自旋轴相关
- 喷流不是从事件视界内部喷出，而是来自黑洞附近的磁场和等离子体环境

### H2：热冕和散射雾

在黑洞周围加入低密度体积光：

- 事件视界外非常淡的高能热冕
- 吸积盘上下方散射薄雾
- bloom 只影响发光区域，不让事件视界发光

### H3：可透镜星场

当前 skybox 被屏幕空间扭曲。后续可升级：

- 程序化高密度星点背景
- 星点在透镜中拉伸成弧
- Debug view 显示采样偏移方向和强度

## 阶段 I：教学系统升级

### I1：讲解脚本模式

增加一组按顺序播放的教学步骤：

1. 事件视界
2. 光子球
3. 吸积盘温度
4. Doppler 增亮
5. 引力透镜
6. 自旋非对称

每一步自动切换 camera preset、parameter preset 和 debug view。

### I2：参数解释面板

每个参数旁边提供简短解释：

- `Spin`：控制旋转方向和非对称强度
- `Lens Strength`：控制屏幕空间透镜偏折量
- `Photon Sphere`：影响亮环半径
- `Disk Thickness`：控制吸积盘边缘厚度感
- `Doppler`：控制朝向相机一侧的蓝移和增亮

### I3：Preset 保存

后续把 UI 中调好的黑洞参数保存到配置文件：

```text
assets/config/black_hole_presets.ini
```

功能：

- 保存当前参数为新 preset
- 从 preset 文件加载
- 恢复默认
- 分享演示配置

## 阶段 J：性能与稳定性

### J1：异步加载

当前启动加载页只是让同步加载可见。后续应做真正异步：

- 资源加载任务队列
- 主线程保持窗口响应
- shader / texture / mesh 加载进度可统计
- 失败资源显示在加载页和日志中

### J2：GPU 兼容层

Intel 集显对复杂 shader 编译更敏感，后续应提供兼容策略：

- shader quality define
- 禁用高复杂噪声循环
- 限制动态分支
- 编译失败自动 fallback 到低质量 shader

### J3：性能面板

Debug UI 中显示：

- 帧率
- draw call 数
- shader reload 状态
- ray steps
- bloom pass 数
- 当前 framebuffer 尺寸

## 阶段 K：场景扩展

黑洞场景稳定后，可以基于现有结构继续扩展：

- 恒星绕黑洞轨道
- 潮汐撕裂事件
- 双黑洞透镜
- 黑洞与星云背景
- 类星体/活动星系核
- 多场景教学目录

场景系统建议继续使用 `SceneCatalog`，每个新场景保持类似接口：

```cpp
update(const Time& time)
draw(Camera& camera, float aspectRatio)
drawControls(Camera& camera)
drawTeachingOverlay(...)
resetCamera(Camera& camera)
```

## 推荐优先级

短期优先：

1. 真正异步加载，解决启动资源加载期间的等待
2. 黑洞 preset 保存/读取
3. 教学脚本模式
4. 透镜质量档
5. 吸积盘前后层分离

中期优先：

1. Schwarzschild ray marching
2. Kerr 自旋近似增强
3. 三维厚度吸积盘
4. 喷流和热冕
5. 星场透镜质量提升

长期优先：

1. 双黑洞或潮汐撕裂事件
2. 更完整的教学课程模式
3. 截图/录制/演示脚本
4. 多场景宇宙渲染框架

## 风险点

- 完整物理精确度和实时性能会冲突，需要质量档和 fallback
- 透明吸积盘、bloom、透镜 pass 顺序容易造成过曝或遮挡错误
- shader 复杂度提升后，Intel 集显和部分旧驱动可能编译失败
- 教学 UI 如果信息过多，会遮挡画面，需要可折叠和步骤化展示
- 后续如果引入多线程加载，OpenGL 资源创建仍要注意上下文归属

## 下一版验收标准

- 黑洞场景能在 `Fast / Balanced / High` 透镜质量之间切换
- 至少一个教学脚本能自动演示事件视界、光子球、Doppler 和透镜
- 参数 preset 可以保存并重新加载
- 吸积盘前景/背面/透镜贡献能在 debug view 中分开查看
- Intel 集显上仍能稳定启动，复杂效果有 fallback
