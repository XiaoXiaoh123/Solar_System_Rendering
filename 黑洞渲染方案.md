# 黑洞渲染方案

## 目标

在现有太阳系渲染项目基础上扩展一个独立的黑洞场景。黑洞场景不应该复用太阳系的行星轨道逻辑，而应该作为新的场景模块接入场景选择系统。

当前项目已经新增 `SceneCatalog`，控制面板顶部可以选择场景，并默认进入 `Solar System`。`Black Hole (planned)` 已在目录中预留，后续实现完成后只需要将其标记为可用，并在主循环中接入对应场景对象。

## 推荐目录结构

## 场景接入方式

1. 新建 `BlackHoleScene`，提供和太阳系类似的生命周期：
   - `update(const Time& time)`
   - `draw(Camera& camera, float aspectRatio)`
   - `drawControls()`
   - `resetCamera(Camera& camera)`

2. 在 `SceneCatalog` 中将黑洞条目改为可用：
   - `Black Hole (planned)` 改名为 `Black Hole`
   - `available = true`

3. 在 `main.cpp` 中根据 `activeScene` 调用对应场景：
   - 太阳系走现有 `SolarSystem`
   - 黑洞走 `BlackHoleScene`

4. 保持共享资源体系：
   - shader 通过 `ResourceManager::getShader()`
   - mesh 通过 `ResourceManager`
   - 纹理和噪声贴图也交给 `ResourceManager`

## 核心视觉组成

### 1. 事件视界

黑洞主体可以用一个纯黑球体或屏幕空间遮罩表示。

关键参数：
```ini
[BlackHole]
massSolar = 4.3e6
schwarzschildRadius = 12.0
spin = 0.6
eventHorizonRadius = 12.0
photonSphereRadius = 18.0
```

视觉要求：
- 事件视界必须完全吸光。
- 边缘可以有非常窄的暗红或蓝白高光，但不能发光成普通星球。
- 视界外应有透镜扭曲，而不是简单黑球。

### 2. 引力透镜

第一阶段建议用屏幕空间近似，不直接上完整广义相对论光线追踪。

做法：
- 先正常渲染星空背景到 HDR framebuffer。
- 在后处理 pass 中，以黑洞屏幕坐标为中心，对背景采样坐标做径向偏移。
- 距离事件视界越近，采样偏移越强。
- 在光子球附近加入环状亮化，模拟 Einstein ring。

近似公式：
```glsl
vec2 dir = uv - blackHoleCenter;
float r = length(dir);
float bend = lensStrength / max(r * r, 0.001);
vec2 bentUv = uv + normalize(dir) * bend;
```

后续可升级为：
- per-pixel ray marching
- Schwarzschild metric 近似积分
- Kerr spin 造成的非对称拖拽

### 3. 吸积盘

吸积盘是黑洞场景最重要的可读视觉元素。

推荐实现：
- 用一个高细分圆环 mesh。
- 顶点位于 XZ 平面，使用内半径和外半径生成。
- fragment shader 中用径向噪声和角向噪声控制亮度。
- 内圈偏蓝白，外圈偏橙红。
- 盘面靠近黑洞的一侧被透镜弯曲到上方，形成经典“上下双弧”视觉。

参数：
```ini
[AccretionDisk]
innerRadius = 18.0
outerRadius = 85.0
temperatureInner = 1.0
temperatureOuter = 0.25
turbulence = 0.55
rotationSpeed = 0.35
dopplerBoost = 0.45
```

### 4. 多普勒增亮

吸积盘绕黑洞高速旋转，朝向相机运动的一侧更亮、更蓝，远离相机的一侧更暗、更红。

近似：
```glsl
float doppler = dot(tangentVelocity, viewDir);
float boost = pow(max(0.0, 1.0 + doppler * strength), 3.0);
color *= boost;
```

这一步会显著提升真实感，即使物理是近似的，也会让画面从“发光圆盘”变成“高速等离子体盘”。

### 5. 星场背景

黑洞需要高对比星场来体现透镜。

建议：
- 复用当前 skybox。
- 增加程序化星点 pass 或高分辨率星图。
- 在透镜 pass 中扭曲背景采样。

## 渲染顺序

推荐顺序：

1. 渲染星空背景到 HDR scene texture。
2. 渲染吸积盘到 HDR scene texture。
3. 渲染事件视界遮罩。
4. 执行引力透镜后处理。
5. 执行 bloom/HDR composite。
6. 绘制 UI 标签和信息面板。

## UI 控制

黑洞场景控制面板建议包含：

- Black Hole Mass
- Spin
- Lens Strength
- Event Horizon Radius
- Photon Sphere Radius
- Accretion Disk Inner Radius
- Accretion Disk Outer Radius

- Lens Field
- Disk Temperature

## 相机交互

黑洞场景不适合复用天体跟随列表，但可以保留同样的交互精神：

- `Focus Black Hole`
- `Orbit Camera`
- `Free Fly`
- `Disk Edge View`
- `Top Down View`


## 实施阶段

### 阶段 A：场景骨架

- 新建 `BlackHoleScene`
- 在场景选择中启用黑洞
- 渲染一个黑色事件视界球体
- 添加基本 UI 参数

### 阶段 B：吸积盘

- 生成圆环 mesh
- 添加温度渐变 shader
- 添加旋转动画
- 接入 bloom

### 阶段 C：屏幕空间透镜

- 后处理 pass 获取黑洞屏幕坐标
- 扭曲背景和吸积盘采样
- 添加光子球亮环

### 阶段 D：真实感提升

- 多普勒增亮
- 噪声湍流
- 盘面厚度
- 远近侧遮挡和弯曲近似

### 阶段 E：教学信息

- 标签和说明面板
- 参数 preset
- Debug view
- 镜头 preset

## 风险点

- 屏幕空间透镜容易在屏幕边缘采样越界，需要 clamp 或 fallback。
- 吸积盘透明混合和 bloom 容易过曝，需要 HDR 参数独立调节。
- 如果后续做真实 ray marching，性能压力会明显上升，需要低/中/高质量档。
- Kerr 黑洞旋转的物理近似复杂，第一版不要直接追求完整精确。

## 推荐第一版验收标准

- 场景选择能切换到黑洞。
- 黑洞事件视界始终为纯黑。
- 吸积盘有内热外冷的颜色变化。
- 背景星场在黑洞周围发生可见弯曲。
- Bloom 只增强吸积盘和光子环，不让事件视界发光。
- UI 能实时调整 lens strength、disk radius、spin、doppler boost。
